## :thinking_face: 型って何？

<!-- block-start: grid -->
<!-- block-start: column -->

```tsx
const str: string = "this is string value";
const num: number = 12345;
const bool: boolean = true;
```

<!-- block-end -->
<!-- block-start: column, left -->

- string / number / boolean などの、データの種類
- 動的型付け言語と静的型付け言語がある
  - 動的: 変数・関数・クラスの定義時に、型を明示的に宣言しない
    - Ruby/JavaScript など
  - 静的: 変数・関数・クラスの定義時に、型を明示的に宣言しなければならない
    - C++/Go/Java など

## TypeScript の登場

<!-- block-start: grid -->
<!-- block-start: column -->

```ts
const user = {
  firstName: "Angela",
  lastName: "Davis",
  role: "Professor",
};

console.log(user.name);
// => Property 'name' does not exist on type '{ firstName: string; lastName: string; role: string; }'.
```

https://www.typescriptlang.org/

<!-- block-end -->
<!-- block-start: column, left -->

- JavaScript の文法はそのままに、型を追加した JavaScript の superset
- build 時に JavaScript に変換する
- ECMAScript 2015 が策定される前に、import/export などの文法を先取って実装
- どのぐらい型の検査を厳しくするかは config で設定できる
  - 0 ⇒ 1 なので厳しい config にしたり
  - JavaScript で数年開発した project に後から TypeScript を入れるのでゆるい config にしたり
  - (余談) React.js では、React.propTypes ⇒ flowtyped ⇒ TypeScript のように型付けのでデファクトが変化していったが、今回は割愛

## ここまでのまとめ

- フロントエンド・JavaScript の開発体験を良くする動きが複数の観点で行われてきた
  - 結果純粋な HTML/CSS/JavaScript では済まない、重厚なスタックで開発を行うようになっている
- JavaScript が誕生した当時には存在しなかった module の概念を実現するために、module bundler が利用されている
- 生の JavaScript では表現しきれない UI を実現するために、React.js をはじめとしたライブラリが利用されている
- 複雑化する JavaScript アプリケーション開発のため、型の恩恵にあずかれる TypeScript が利用されている
