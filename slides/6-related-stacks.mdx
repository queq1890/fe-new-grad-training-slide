# 他にどんな知識が必要？ :thinking_face:

---

## React.js を使ったプロジェクトで考慮すること

- 環境をどう作るか (Next.js / Vite / Webpack...)
- Linter/Formatter
- リポジトリのディレクトリ構成
- API との通信部分をどう実装するか
- どの rendering 方法を使うか (CSR / SSR / SSG / ISR / Server Component... )
- State 管理の方法 (React.js 標準の hooks を使うか、外部のライブラリを使うか)
- コンポーネントのスタイリング
- フォーム・バリデーションの実装方法
- TypeScript の設定をどのぐらい厳しくするか
- Storybook を採用するか
- 何をテストするか
- どこにデプロイするか
- SEO 対策
- BFF (Backend For Frontend) server を用意するか
- 他ロールとどうやって協業するか (デザイナー・バックエンドエンジニア...)

<br />

## React.js を使ったプロジェクトで考慮すること

- 色々ある！ :cry:
- プロジェクトの規模・メンバーの習熟度などを考慮して、技術選定する
  - 完全に同じスタックのプロジェクトは生まれにくい
- Q. React.js に関連した技術郡はどうやって勉強すれば良い？
- Q. 技術選定に迷ったら、何を参考にすれば良い？

## Bulletproof React

https://github.com/alan2207/bulletproof-react

- React.js を使ったプロジェクトの模範的な実装例としてよく挙げられるリポジトリ
- 各技術スタックの採用理由が docs に書いてあるので、技術選定に迷った時に眺めてみるといいかも
